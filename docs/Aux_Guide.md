# Guia Auxiliar

## Nomeação de Arquivos e Pastas

1. Use nomes descritivos e padronizados:
   - Pastas e arquivos devem ter nomes em inglês ou no idioma do projeto.
2. Separe por tipo ou propósito:
   - Centralize imagens, diagramas ou outros anexos em uma pasta como `assets/`.
3. Inclua sumários nos arquivos principais.

## Dicas Adicionais

- Automatize tarefas repetitivas: Use ferramentas como Makefiles ou scripts para gerar relatórios, compilar o projeto, etc.
- Documente dependências: Inclua informações claras sobre bibliotecas, frameworks e versões usadas no arquivo `README.md` ou `Deployment.md`.
- Versionamento: Utilize Git para versionar tanto o código quanto a documentação.
  - Use uma branch separada para a documentação (`docs/`) caso necessário.
- Integração com Ferramentas de Gestão:
  - Considere usar ferramentas como o Notion, Confluence ou Google Drive para armazenar documentos colaborativos.
  - Utilize o GitHub Wiki para hospedar a documentação diretamente no repositório.
